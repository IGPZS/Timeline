## 4. АВТОМАТИЗАЦИЯ (GitHub Actions)

### get-images.yml
- Загружает изображения напрямую с `upload.wikimedia.org`
- Фильтрует файлы меньше 5000 байт (во избежание заглушек)
- Сохраняет в `assets/images/`
- Коммитит и пушит изменения автоматически

### fix-image-paths.yml
- Исправляет пути в `EV_IMGS` в `index.html`
- Заменяет внешние URL на локальные `assets/images/...`

## 5. ИЗМЕНЕНИЯ В index.html

### EV_IMGS
Массив объектов с ключами, соответствующими заголовкам событий в массиве `EV`:
```js
const EV_IMGS = {
  "Троянская война": "assets/images/troy.jpg",
  // ... 16 остальных
};
```

### Функция setImage
- Создаёт элемент `<img>` через DOM
- **Сначала** устанавливает `onerror` (избегает race condition)
- **Затем** устанавливает `src`
- При ошибке показывает текст «Изображение недоступно»

### CSS изменения
- Десктоп: `.tl-img { height: 260px }`
- Мобильный: `.tl-img { height: 200px }`

## 6. ОШИБКИ И ИСПРАВЛЕНИЯ

| Ошибка | Причина | Решение |
|--------|---------|----------|
| Изображения не загружаются | Wikimedia `Special:FilePath` URL не работают напрямую | Заменить на прямые URL `upload.wikimedia.org` |
| Race condition в setImage | `onerror` назначался после `src` | Назначить `onerror` до установки `src` |
| Пустой тег `<img>` в шаблоне | Браузер пытается загрузить пустой URL | Убрать пустой тег, рендерить через `setImage` |
| Изображения слишком маленькие | Высота контейнера не задана | Добавить фиксированную высоту в CSS |
